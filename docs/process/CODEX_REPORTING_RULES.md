# Codex reporting rules

## Обязательный отчет Codex

Каждый этап должен иметь отчет с понятным статусом и проверяемыми доказательствами.

## Обязательные разделы

- `status`: `ok`, `ok_with_warnings`, `needs_hotfix`, `failed`.
- `summary`: короткое описание результата.
- `root cause`: если исправлялся баг.
- `changed files`: измененные файлы.
- `created files`: созданные файлы.
- `read / inspected files`: прочитанные файлы.
- `commands run`: команды, которые запускались.
- `tests`: проверки и результаты.
- `not run`: что не запускалось и почему.
- `safety confirmations`: подтверждения безопасности.
- `warnings`: предупреждения.
- `limitations`: ограничения.
- `next step`: рекомендуемый следующий шаг.

## Требования к точности

- Не писать "готово", если часть критериев не выполнена.
- Если проверка не запускалась, указать причину.
- Если GitHub, CI, browser, network или external tool недоступны, указать это явно.
- Не скрывать `ok_with_warnings`, если есть предупреждения.

## Safety confirmations

В каждом финальном отчете подтвердить:

- пользовательские документы не изменялись;
- пользовательские документы не добавлялись в репозиторий;
- runtime / cache / temp / downloads не добавлялись;
- secrets не читались и не добавлялись;
- live network не запускался без прямого разрешения;
- browser automation не запускалась без прямого разрешения;
- EXE / build не создавался без прямой команды;
- изменения ограничены scope этапа.
