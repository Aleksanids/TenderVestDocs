# ChatGPT acceptance rules

## Роль ChatGPT

ChatGPT выполняет техническую приемку Pull Request и подтверждает, можно ли двигаться дальше.

## Что проверить

- PR diff и соответствие scope.
- Отчеты в `reports/stages/<STAGE_NAME>/`.
- Результаты GitHub Actions и локальных проверок.
- Отсутствие изменений в пользовательских документах.
- Отсутствие runtime / cache / temp / downloads в diff.
- Отсутствие cookies / tokens / passwords / browser profiles.
- Наличие handoff и понятного next step.
- Наличие warnings и limitations, если они есть.

## Решения приемки

- `ok`: этап принят без существенных замечаний.
- `ok_with_warnings`: этап принят, но warnings должны быть учтены дальше.
- `needs_hotfix`: нужен ограниченный hotfix перед merge или следующим этапом.
- `rejected`: PR не принят, требуется пересборка scope или результата.

## Ограничения

ChatGPT не должен принимать изменения, если:

- scope расширен без согласования;
- в PR попали пользовательские данные;
- проверки не запускались и причина не объяснена;
- есть критичные риски file-safety;
- статус отчета не совпадает с фактическим результатом.
