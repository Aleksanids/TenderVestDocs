# GitHub-контроль TenderVestDocs

GitHub используется как центральное место контроля разработки.

## Порядок

1. Один этап = одна ограниченная задача.
2. Для этапа создается отдельная ветка.
3. Изменения оформляются через Pull Request.
4. В Pull Request должен быть отчет: цель, изменения, проверки, ограничения, статус.
5. До приемки изменения не считаются принятыми.

## Решения приемки

- ok
- ok_with_warnings
- needs_hotfix
- rejected
