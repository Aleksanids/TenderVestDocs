# TenderVestDocs

Рабочий репозиторий проекта TenderVestDocs.

## Модель контроля

Разработка ведется через GitHub: отдельная задача, отдельная ветка, Pull Request, отчет этапа, автоматическая safe/offline проверка, кодовый аудит и приемка ChatGPT / Claude / пользователя.

## Базовые решения

- Один этап Codex = одна ограниченная задача.
- Основная ветка не должна изменяться без приемки.
- Пользовательские документы и тяжелые выгрузки не хранятся в репозитории.
- Отчеты этапов хранятся в `reports/stages/`.
- Процесс описывается в `docs/process/`.

## GitHub workflow

Стандартный цикл этапа:

1. Codex получает prompt и фиксирует scope.
2. Codex создает отдельную ветку.
3. Codex вносит минимальные изменения.
4. Codex запускает локальные проверки.
5. Codex создает отчет в `reports/stages/<STAGE_NAME>/`.
6. Codex открывает Pull Request.
7. GitHub Actions запускает `codex_audit`.
8. ChatGPT / Claude / пользователь проводят review.
9. Merge допускается только после решения `ok` или `ok_with_warnings`.

Подробные правила находятся в `docs/process/`.
