# TenderVestDocs

Рабочий репозиторий проекта TenderVestDocs.

## Модель контроля

Разработка ведется через GitHub: отдельная задача, отдельная ветка, Pull Request, отчет этапа, автоматическая проверка, кодовый аудит и приемка ChatGPT.

## Базовые решения

- Один этап Codex = одна ограниченная задача.
- Основная ветка не должна изменяться без приемки.
- Пользовательские документы и тяжелые выгрузки не хранятся в репозитории.
- Отчеты этапов хранятся в `reports/stages/`.
- Процесс описывается в `docs/process/`.
